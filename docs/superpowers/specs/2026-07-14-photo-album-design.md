# 云端相册设计规格

## 目标

构建一个兼容手机端和网页端的 React 云相册。访客可以打开公开相册并翻页浏览照片；固定管理员登录后可以管理相册、批量上传图片并调整照片展示方式。

## 技术方案

- 前端使用 React 和 TypeScript，采用响应式布局。
- 应用部署在 Sites 平台。
- Cloudflare D1 保存管理员、相册、照片元数据和会话信息。
- Cloudflare R2 保存照片文件。
- 服务端 API 统一执行鉴权、输入校验、数据读写和文件操作。
- 浏览器存储仅用于界面偏好，不作为相册或照片的权威数据源。

## 用户与权限

系统仅有一个固定管理员账号。管理员邮箱和密码在部署环境中配置，密码只保存安全哈希结果，不进入前端代码或版本库。

- 访客可以查看公开相册列表和相册中的照片。
- 访客不能上传、修改、排序或删除任何内容。
- 管理员登录后可以创建、改名、删除相册，批量上传、排序、设置封面和删除照片。
- 管理接口必须验证服务端会话；登录 Cookie 使用 `HttpOnly`、`Secure`、`SameSite` 和有效期限制。

## 访客端

### 相册首页

- 采用“温暖家庭相册”视觉方向：暖米色背景、轻微纸张质感、陶土色与棕红色实体相册封面。
- 每张相册卡片显示封面、名称、照片数量和最近更新时间。
- 相册按更新时间倒序排列。
- 点击相册进入沉浸式浏览页。
- 没有相册时展示友好的空状态。

### 翻页浏览

- 桌面端一次展示左右两页，具有克制的翻书动画。
- 支持点击页角、上一页/下一页按钮和键盘左右方向键。
- 手机端自动切换为单张展示，支持左右滑动和按钮切换。
- 页面显示相册标题、当前照片位置、总照片数和返回入口。
- 照片保持原始比例完整显示，不强制裁切；多余区域使用纸张留白。
- 翻页时预加载相邻照片，其他照片使用懒加载。
- 图片加载失败时显示占位状态和重试操作。

## 管理端

### 登录

- 管理员通过固定邮箱和密码登录。
- 登录失败使用统一错误提示，不暴露账号是否存在。
- 会话过期后返回登录页，并提示重新登录。

### 相册管理

- 新建相册时填写相册名称。
- 相册可以改名、设置封面和删除。
- 首张成功上传的照片默认成为封面，管理员可以重新指定。
- 删除相册前显示二次确认，并说明其中照片也会永久删除。

### 批量上传

- 上传流程为：创建或选择相册、选择多张图片、检查上传队列、开始上传。
- 支持 JPG、PNG、WebP 和 GIF。
- 单张文件最大 20MB；前端先行校验，服务端再次校验文件类型和大小。
- 上传队列显示缩略图、文件名、进度、成功或失败状态。
- 多张图片采用有限并发逐张上传；单张失败不阻断其他图片。
- 失败项目保留在队列中，并支持单独重试。

### 照片管理

- 照片可以拖动排序、设为封面和删除。
- 删除照片前要求确认。
- 删除封面照片后，系统自动选择排序最前的剩余照片作为新封面；空相册不显示封面。

### 响应式布局

- 电脑端后台采用相册列表与照片管理双栏布局。
- 手机端使用单列卡片布局和便于拇指操作的底部操作区。
- 所有交互控件满足触摸尺寸和键盘可访问性要求。

## 数据模型

### albums

- `id`: 相册唯一标识。
- `name`: 相册名称。
- `cover_photo_id`: 可空的封面照片标识。
- `created_at`: 创建时间。
- `updated_at`: 更新时间。

### photos

- `id`: 照片唯一标识。
- `album_id`: 所属相册标识。
- `object_key`: R2 文件键。
- `original_name`: 原文件名。
- `content_type`: 文件类型。
- `size_bytes`: 文件大小。
- `sort_order`: 展示顺序。
- `created_at`: 上传时间。

### admin_sessions

- `id`: 会话唯一标识。
- `token_hash`: 会话令牌哈希。
- `expires_at`: 过期时间。
- `created_at`: 创建时间。

管理员邮箱和密码哈希通过部署环境提供，不在 D1 中维护多用户账号体系。

## API 边界

- 公开接口：获取相册列表、获取指定相册及其照片、读取照片文件。
- 管理接口：登录、退出、新建/改名/删除相册、批量上传、删除照片、设置封面和保存照片顺序。
- 所有写接口执行会话校验和请求校验。
- R2 对象不直接开放写权限，上传必须通过受保护的服务端接口。

## 一致性与错误恢复

- 创建照片时先上传 R2，再写入 D1；D1 写入失败时删除刚上传的 R2 对象。
- 删除照片时先标记或删除 D1 记录，再删除 R2 对象；对象清理失败记录服务端错误，避免向访客继续展示已删除照片。
- 删除相册时批量移除照片记录和相册记录，然后清理对应 R2 对象。
- 网络中断、文件不合法、容量不足和会话失效均返回明确的中文错误信息。
- 上传队列保留失败项，允许管理员重试，不自动重复已成功上传的文件。

## 测试与验收

- 单元测试覆盖文件校验、翻页索引、封面回退和照片排序。
- API 测试覆盖登录、权限拦截、相册增删改、上传失败回滚和删除流程。
- 组件测试覆盖上传队列、空状态、错误状态及桌面/手机翻页控制。
- 构建必须无 TypeScript 错误。
- 桌面端验收双页翻书、键盘控制和管理双栏布局。
- 手机端验收单张滑动、触摸按钮、上传队列和管理单列布局。

## 首版范围

首版不包含用户注册、家庭成员邀请、评论、点赞、公开分享口令、视频上传、自动人脸识别、AI 分类或照片编辑。首版只实现已确认的公开浏览与单管理员完整管理能力。
